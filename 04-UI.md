# 04-UI.md

# Pixel Pond UI Design Specification

**Version:** 2.0 (Web Application Edition)

------------------------------------------------------------------------

# 1. Design Philosophy

Pixel Pond is a **web application**, not a full-screen browser game.

The interface should combine:

-   Modern dashboard layout
-   Cozy pixel-art visuals
-   Clean UX
-   Responsive design
-   Fast interactions

The website should feel like a management dashboard with game elements
rather than a traditional RPG interface.

------------------------------------------------------------------------

# 2. Visual Style

Theme:

-   Cozy
-   Relaxing
-   Friendly
-   Pixel-art inspired

Visual references:

-   Stardew Valley
-   Animal Crossing
-   GitHub Dashboard
-   Steam Library

------------------------------------------------------------------------

# 5. Page Specifications

## Login

Purpose

Authenticate users.

Content

-   Logo
-   Welcome message
-   GitHub Login button
-   Pixel-art lake background

------------------------------------------------------------------------

## Dashboard

Purpose

Provide an overview of player progress.

Cards

-   Gold
-   Equipped Rod
-   Collection Progress
-   Today's Catch

Sections

-   Recent Catches
-   Recent Transactions

Primary Action

-   Go Fishing

------------------------------------------------------------------------

## Fishing

Purpose

Perform a fishing action.

Layout

Top

-   Current Rod
-   Luck Bonus

Center

-   Pixel lake illustration

Primary Button

-   Go Fishing

Result

-   Modal displaying fish image, rarity and sell value.

------------------------------------------------------------------------

## Inventory

Purpose

Manage owned fish.

Display

-   Responsive card grid

Each card contains

-   Fish image
-   Fish name
-   Quantity
-   Sell button

------------------------------------------------------------------------

## Collection

Purpose

Track discovered fish.

Display

-   Grid layout

Unknown fish remain hidden until discovered.

Filters

-   All
-   Common
-   Rare
-   Epic
-   Legendary

------------------------------------------------------------------------

## Shop

Purpose

Purchase fishing rods.

Each rod card displays

-   Image
-   Name
-   Price
-   Luck Bonus

Actions

-   Buy
-   Equip

------------------------------------------------------------------------

## Market

-   Sell button

# 6. Responsive Rules

Desktop

-   Sidebar + Content

Tablet

-   Drawer navigation

Mobile

-   Bottom navigation
-   One-column layout

------------------------------------------------------------------------

# 7. Loading States

Use Skeleton Loading for:

-   Dashboard
-   Inventory
-   Collection
-   Shop

Buttons display loading indicators while Server Actions execute.

------------------------------------------------------------------------

# 8. Error States

Provide friendly error messages.

Examples

-   Failed to load data.
-   Unauthorized.
-   Insufficient Gold.
-   Something went wrong.

------------------------------------------------------------------------

# 9. Empty States

Examples

-   No fish collected.
-   No catches yet.
-   No rods purchased.

------------------------------------------------------------------------

# 10. Animations

Use lightweight animations only.

-   Fade
-   Scale
-   Hover
-   Button Press

Avoid complex game animations.

------------------------------------------------------------------------

# 11. Asset Guidelines

All graphics should use a consistent pixel-art style.

Assets include

-   Fish sprites
-   Rod icons
-   Logo
-   Lake background
-   UI icons
-   Badges

Store assets under:

``` text
/public/assets/
```

Use Next.js Image component for all images.

------------------------------------------------------------------------

# 12. UI Principles

-   Mobile-first
-   Consistent spacing
-   Reusable components
-   Accessible
-   Fast loading
-   Production-ready
-   Pixel visuals with modern web layout
